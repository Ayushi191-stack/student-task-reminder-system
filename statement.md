# Student Task Reminder System - Project Statement
1.Problem Statement
⁘ students often forget important academic tasks such as assignments,homework, exam schedules and project deadlines.This leads to last-minute stress and poor time mangement. A simple and organised task reminder system is needed to help students store,view and manage their daily tasks effectively.
 students often forget important academic tasks such as assignments,homework, exam schedules and project deadlines.This leads to last-minute stress and poor time mangement. A simple and organised task reminder system is needed to help students store,view and manage their daily tasks effectively.

2.Scope of the Project
⁘ The project covers baic task management functionalities:
  • Adding new tasks
  • Viewing existing tasks
  • Updating task details
  • Deleting tasks
  • Marketing tasks as completed
⁘ Out of scope:
  • Cloud backup
  • Push notifications
  • Multi-user login system
  (These can be added in future versions.)
3.Target Users
  • School students
  • College students
  • Anyone who needs help remembering daily study tasks and deadlines

4.Objectives
  • To create a simple and user-friendly task reminder system
  • To help students keep track of daily assignments and deadlines
  • To allow students to easily add,view,update and delete tasks
  • To storee all tasks safely in a file or local storage
  • To support better time management and productivity

5.High-level Features
  • Add a new task with title,description,date and time
  • View all tasks
  • View today's tasks
  • Edit or update an existing task
  • Delete a task
  • Mark a task as completed

6.Functional Requirements
⁘ Task Input Module
  • Add task
  • Edit task
  • Validate task details(date,time,etc.)
⁘ Task Management Module
  • Save tasks
  • Update tasks
  • Delete tasks
  • Mark task as completed
⁘ Task Display Module
  • View all tasks
  • View today's tasks
  • show completed tasks
  • Show pending tasks

7.Non-Functional Requirements
 • **Usability:**Simple and easy-to-use interface
 • **Performance:** should respond within 1-2 seconds
 • **Reliability:** Task data should not be lost
 • **Error Handling:** Proper messages for invalid input
 • **Maintainbility:** Code should be modular and easy to update
 • **Resource Efficiency:** Lightweight and efficient for any system
